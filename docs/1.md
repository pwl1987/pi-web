## 2) UI 优化建议

### 2.1 信息层级 —— 让"讨论 / 方案 / 任务"三级清晰分层

- **顶栏加状态条**：在 `AppShell.tsx:1024-1095` 区域新增一个 `PlanModeBanner`（仅 `planMode === true` 时渲染），固定贴在顶栏下方 1px，背景用 `--color-doc-bg`，左侧 8px 圆点用 `var(--accent)` 呼吸动画（复用 `InspectorTaskRow.tsx` 的 `inspector-pulse` keyframes），文案按 `planStatus` 切换：
  - `parsing` / `discussing` → `t("plan.statusRunning", { round, max })`
  - `awaiting_confirm` → `t("plan.statusNeedsApproval")`（强调色 `--color-warning`）
  - `executing` → `t("plan.statusHandoff")`（强调色 `--git-added`）
  - `done` / `failed` / `cancelled` → 各自语义色 + 一个 `查看引擎`/`返回普通` 按钮
  - 这样用户**任何时刻**都能在屏幕顶端看到当前所处阶段与下一步操作，不必滚到 PlanPanel 底部。
- **PlanPanel 内三层用统一 `<header>` + `<section>` 语义**：
  - `components/PlanPanel.tsx:330-440` 的「配置区/角色条/日志切换/时间线」四段在视觉上没有层级差，改为：
    - 一级：状态头（带进度环 + 状态徽标 + 退出）
    - 二级：参与角色条（avatar + thinking/responded 状态微动效）
    - 三级：可折叠的「配置 / 日志 / 历史」二级抽屉
  - 关键改动：把分散的 `borderBottom: "1px solid var(--border)"` 提取到顶层 `<div>`，避免每行都画线导致"信息密度塌陷"。

### 2.2 视觉焦点 —— 等待审批时把"方案卡"做成主视觉

- `PlanPanel.tsx:452-518` 的方案卡当前是 1px border 的小卡片，点击仅切换 `selectedPlanId`。**改为**：
  - 默认卡片用 `var(--bg)` + 1px `var(--border)`；
  - 选中后用 `--accent-bg` + 2px `var(--accent)` + 左侧 3px 实心条（`borderLeft: 3px solid var(--accent)`）；
  - 顶部加 confidence 进度条（细 4px 高），颜色按 `confidence >= 0.8 → --git-added`、`< 0.6 → --color-error-soft`，**复用** `InspectorPanel.tsx:1362-1412` 已有 `ProgressRing` 的实现思路（或直接内联 CSS `linear-gradient`），不引入新组件。
- **方案标题旁加状态徽标**：从 `RecommendationPlan` 里推导（如果有 `riskLevel` 字段）或用 confidence 阈值（≥0.8 安全 / 0.6-0.8 谨慎 / <0.6 风险），三个色板对齐 `globals.css:40-58` 已有的 `--color-error-soft / --color-success-soft / --color-warning` 语义色，**避免**继续在 `PlanTaskCard.tsx:8-14` 那种硬编码 `#22C55E / #EF4444 / #F59E0B`。

### 2.3 状态区分 —— 用一致的 5 状态色板替换散落硬编码

- 收敛所有 plan/task 状态色到 `globals.css` 的语义 token（已存在 `--git-added / --git-deleted / --color-error-soft / --color-warning / --color-success-soft`），落到 `PlanTaskCard.tsx:8-14` 与 `AutonomousCodingDashboard.tsx:21-29` 同一个映射表：
  - pending → `var(--text-dim)` / running → `var(--accent)` / completed → `--git-added` / failed → `--color-error-soft` / skipped → `--color-warning`
- 状态机 5 个阶段徽标复用 `InspectorTaskRow.tsx:71-103` 的 `statusDot` 模式（圆点+内勾/呼吸），新建 `components/PlanStatusDot.tsx`（约 30 行）统一所有 plan UI 的状态点；这是 Ponytail 原则"已经有 InspectorTaskRow 的实现就别再写一遍"。
- **`executing` 阶段加 skeleton loading**：`AutonomousCodingDashboard.tsx:262-281` 在 `selected` 还没拿到任务列表时只有一个空文本，**替换为** `components/Skeleton.tsx` 已有原语（如果是行级占位就用其 row 模式）。这样从 plan → engine 的交接瞬间不会"白屏一闪"。
